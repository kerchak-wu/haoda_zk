# 好搭掌控双板温湿度语音播报系统

## 一、项目概述

本项目基于两块"好搭掌控"开发板，实现一套无线温湿度采集与语音播报系统。

- **掌控1**：通过 DHT11 传感器采集温湿度，OLED 显示数据，并通过 radio 广播发送
- **掌控2**：通过 radio 接收温湿度数据，OLED 显示数据，响应语音命令播报温湿度

两板之间采用掌控板原生 `radio` 模块进行无线通信，**无需 WiFi、无需路由器、无需 MQTT 服务器**，开机即可工作。

---

## 二、系统架构

```
┌──────────────────────────┐         radio 广播          ┌──────────────────────────────┐
│   好搭掌控1 (采集端)      │   channel=5, 2秒/次         │   好搭掌控2 (语音端)          │
│                          │   "温度=xx"                 │                              │
│  P0 ── DHT11             │ ──────────────────────────> │  Timer13 (20ms)              │
│  OLED 显示温湿度         │   "湿度=xx"                 │  radio.receive() 接收         │
│  Timer13 (1秒) 采样      │ ──────────────────────────> │                              │
│  radio.send() 广播       │                             │  P15 ── 语音识别模块 2.0      │
│                          │                             │  P16 ── 语音合成模块 V2.1     │
│                          │                             │  OLED 显示温湿度             │
│                          │                             │  语音命令触发 TTS 播报        │
└──────────────────────────┘                             └──────────────────────────────┘
```

### 通信协议

采用掌控板官方示例的 `"键=值"` 文本协议，频道 5：

| 消息内容     | 含义           |
|:------------|:---------------|
| `温度=25`   | 当前温度 25℃   |
| `湿度=60`   | 当前湿度 60%RH |

掌控1 每 2 秒广播一次，分别发送温度和湿度两条消息。

## 五、核心功能说明

### 5.1 掌控1（采集端）

[controller1_dht11_publish.py](file:///d:/笔记本同步/好搭掌控资料/controller1_dht11_publish.py)

**主要功能：**
1. Timer13 每秒触发 DHT11 采样，更新全局温湿度变量
2. 主循环每 2 秒通过 `radio.send()` 广播 `温度=xx` 和 `湿度=xx`
3. OLED 每 500ms 刷新一次显示
4. 主循环每次 `time.sleep_ms(10)` 让出 CPU

**关键参数：**
- DHT11 采样周期：1000ms
- radio 广播周期：2000ms
- OLED 刷新周期：500ms
- radio 频道：5

### 5.2 掌控2（语音端）

[controller2_voice_subscribe.py](file:///d:/笔记本同步/好搭掌控资料/controller2_voice_subscribe.py)

**主要功能：**
1. Timer13 每 20ms 调用 `radio.receive()` 取广播消息
2. `radio_recv()` 解析 `键=值` 格式更新温度/湿度变量
3. 主循环检查 ASR 是否有新命令
4. 收到命令后调用 TTS 播报，OLED 同步更新
5. OLED 每 500ms 刷新一次显示

**关键参数：**
- radio 接收周期：20ms
- ASR 防抖延迟：500ms
- TTS 音量：9（范围 0-9）
- OLED 刷新周期：500ms

### 5.3 数字转中文读法

为解决 TTS 将 `50` 读成"五零"而非"五十"的问题，[controller2_voice_subscribe.py](file:///d:/笔记本同步/好搭掌控资料/controller2_voice_subscribe.py#L42-L61) 中实现了 `数字转中文()` 函数：

| 数值 | 普通读法 | 修正后读法 |
|:-----|:---------|:-----------|
| 0    | 零       | 零         |
| 9    | 九       | 九         |
| 10   | 一零     | 十         |
| 15   | 一五     | 十五       |
| 50   | 五零     | 五十       |
| 99   | 九九     | 九十九     |
| 100  | 一零零   | 一百       |

支持范围：0-100。超出范围回退为原始数字字符串。

## 七、部署步骤

### 1. 烧录 ASR 语音识别模块
使用 ASRPRO 软件烧录以下词条到语音识别模块 2.0：
- 唤醒词：智能管家
- 命令词：当前温度（ID=53）
- 命令词：当前湿度（ID=54）

### 2. 接线

**掌控1：**
- DHT11 数据线 → P0

**掌控2：**
- 语音识别模块 2.0 TX → P15（掌控2 RX）
- 语音合成模块 V2.1 → P16

### 3. 烧录程序
- 将 `controller1_dht11_publish.py` 烧录到掌控1
- 将 `controller2_voice_subscribe.py` 烧录到掌控2

### 4. 上电运行
1. 两块板同时上电
2. 掌控1 OLED 显示"广播发送中..."，温湿度开始刷新
3. 掌控2 OLED 显示"radio监听中..."，温湿度同步更新
4. 对着掌控2说"智能管家"唤醒
5. 听到提示后说"当前温度"或"当前湿度"，语音合成模块播报

## 八、常见问题

### Q1: 掌控2 收不到温湿度数据？
- 检查两块板 radio 频道是否一致（当前都是 `channel=5`）
- 检查两块板距离是否过远（建议 10 米以内）
- 通过串口查看掌控1 是否有 `Broadcast Failed:` 报错

### Q2: 语音播报声音异常或没有声音？
- 检查语音合成模块 V2.1 接线（P16）和电源
- 检查 `syn.volume(9)` 音量设置
- 检查 ASR 模块是否正确烧录了命令词

### Q3: 语音命令没有响应？
- 确认 ASR 模块接线正确（P15）
- 确认 UART 波特率为 115200
- 确认 ASR 模块已烧录唤醒词和命令词
- 命令词 ID 必须与程序中的判断一致（温度=53，湿度=54）

### Q4: 温湿度读数为 0？
- 检查 DHT11 接线（P0）
- DHT11 采样失败会在程序中静默跳过，温湿度保持上一次值或初始 0
- 重启掌控1 观察 OLED 是否更新

### Q5: 数值读法不对？
- 确认 `数字转中文()` 函数正常工作
- 当前支持 0-100 范围，超出范围会读原始数字（如 105 读"一零五"）
