# 触摸键控制板载RGB灯 — 项目说明

## 一、项目简介

本项目运行于 **掌控板（mPython）**，使用板载 6 个电容触摸键（P / Y / T / H / O / N）对板载 3 颗 RGB 灯进行实时控制，并在 OLED 屏幕上显示当前状态。

- **硬件平台**：掌控板（ESP32，含板载 OLED、RGB、触摸焊盘）
- **开发环境**：MicroPython + `mpython` 库
- **文件名**：`touch_rgb_control.py`

## 二、硬件资源

| 资源 | 说明 |
| --- | --- |
| 触摸键 P / Y / T / H / O / N | 电容触摸焊盘，`touchPad_X.read() < 400` 判定为按下 |
| 板载 RGB 灯 ×3 | 通过 `rgb[0]` / `rgb[1]` / `rgb[2]` 单独控制，或 `rgb.fill()` 全部控制 |
| OLED 显示屏 | 128×64，单行高度 16px，共 4 行有效显示区 |

## 三、触摸键功能

| 按键 | 功能 | 说明 |
| --- | --- | --- |
| **P** | 开关灯 | 切换 `is_on` 状态 |
| **Y** | 切换颜色 | 红→绿→蓝→黄→青→紫→白 循环 |
| **T** | 切换目标灯 | LED1 → LED2 → LED3 → 全部 → LED1 … |
| **H** | 亮度 + | 每次增加 32，上限 255 |
| **O** | 亮度 − | 每次减少 32，下限 0 |
| **N** | 立即熄灭 | 全部 RGB 关闭，`is_on=False` |

## 四、OLED 显示（4 行）

```
触摸键控制RGB灯         <- y=0   标题
目标:LED1 状态:开      <- y=16  目标+状态合并一行
颜色:红                <- y=32  当前颜色
亮度:100%              <- y=48  当前亮度百分比
```

亮度百分比计算：`brightness * 100 // 255`

## 五、核心设计

### 1. 状态变量

| 变量 | 含义 | 取值 |
| --- | --- | --- |
| `current_led` | 当前选中的灯索引 | 0/1/2/3（3=全部） |
| `is_on` | 是否点亮 | True/False |
| `color_index` | 颜色索引 | 0~6 |
| `brightness` | 亮度 | 0~255 |

### 2. 颜色与亮度合成

`apply_led()` 在每次状态变化时被调用，将 **基础颜色 × 亮度比例** 合成最终 RGB 值后写入：

```python
color = ((base[0] * br + 127) // 255,
         (base[1] * br + 127) // 255,
         (base[2] * br + 127) // 255)
```

- `+127` 用于四舍五入，避免低亮度时颜色全黑
- 关闭状态下直接输出 `(0, 0, 0)`

### 3. 触摸键扫描

使用 `Timer(12)` 周期 100ms 轮询 6 个触摸键：

- 每个键独立 `_status_x` 标志位（0=未按，1=按下）
- 仅在状态从 0→1 时触发回调（下降沿），实现单击去抖
- 首次 `touchPad_P.read()` 用于探测 I2C 是否异常，异常时直接返回避免崩溃

### 4. 主循环

```python
while True:
    time.sleep_ms(50)
```

主循环仅做 CPU 让出，实际控制逻辑全部由 Timer 中断驱动，符合掌控板"中断+轮询"的常规编程模式。

## 六、使用方法

1. 用 USB 连接掌控板
2. 通过 IDE（mPython-X / Mu / Thonny）将 `touch_rgb_control.py` 上传到掌控板
3. 运行程序，OLED 显示标题与初始状态
4. 触摸对应焊盘即可控制 RGB 灯：
   - 先按 **T** 选择要控制的灯
   - 按 **P** 开灯
   - 按 **Y** 切换颜色
   - 按 **H / O** 调节亮度
   - 按 **N** 一键熄灭
