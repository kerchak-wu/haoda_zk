# 上海实时天气OLED显示系统

基于掌控板（mpython）的上海实时天气显示程序，连接WiFi后通过心知天气API获取上海实时天气数据，在OLED屏幕上显示天气、温度、风力等信息及实时北京时间。

## 功能特性

- 自动连接指定WiFi（`haoda7`）
- WiFi连接后自动通过NTP同步北京时间（UTC+8）
- 调用心知天气API获取上海实时天气数据：
  - 实时天气状况（晴/雨/多云等）
  - 当前温度
  - 今日最高/最低温度
  - 风向、风力等级
- OLED屏幕4行显示，第4行实时时钟每500ms跳动
- 天气数据每10分钟自动刷新一次
- NTP同步失败时自动尝试备用服务器
- 天气获取异常时保留上次数据显示，不黑屏

本程序**无需外接任何传感器或模块**，仅使用掌控板自带资源：

## 工作原理

### 1. 整体流程

```
开机 → 连接WiFi → NTP同步北京时间 → 首次获取天气 → 显示天气+时钟
                                                    ↓
                                              进入主循环
                                                    ↓
                            ┌──────────────────────┴──────────────────────┐
                            ↓                                              ↓
                    每500ms刷新OLED                                 每10分钟刷新天气
                    （实时时钟跳动）                                （重新拉取API）
```

### 2. 天气数据来源

使用心知天气（Seniverse）免费API：

- **实时天气接口**：`https://api.seniverse.com/v3/weather/now.json`
  - 返回当前温度、天气状况
- **每日预报接口**：`https://api.seniverse.com/v3/weather/daily.json`
  - 返回今日最高/最低温、风向、风力等级
- **城市定位**：`shanghai`（上海）
- **API密钥**：复用项目已有密钥 `S68-rNDesn1uy3ODc`

### 3. NTP时间同步

掌控板开机后RTC未初始化，`time.localtime()` 从 `00:00:00` 开始按开机时长计时，必须显式调用NTP同步：

```python
ntptime.settime(8, 'ntp3.aliyun.com')   # 主服务器
ntptime.settime(8, 'time.windows.com')  # 备用服务器
```

参数 `8` 表示 UTC+8（北京时间/上海时间）。

### 4. OLED显示布局

OLED分辨率 128×64，每行16像素高，分4行显示：

```
┌────────────────────────────────┐
│ 上海实时天气                   │ ← 第1行：标题
│ 天气:晴 28C                    │ ← 第2行：天气状况 + 当前温度
│ 22-30C 东南风                  │ ← 第3行：温度范围 + 风向
│ 风力:3级 14:25:36              │ ← 第4行：风力 + 实时时钟
└────────────────────────────────┘
```

## 使用方法

### 1. 准备工作

- 确保掌控板已通过USB连接电脑
- 确保WiFi环境（`haoda7`）可用
- 安装好搭Block或使用REPL工具

### 2. 烧录程序

将 `shanghai_weather_oled.py` 烧录到掌控板。

### 3. 运行流程

程序运行后，OLED屏幕依次显示：

1. `正在连接WiFi...`（约2-5秒）
2. `同步时间中...`（NTP同步，约1-3秒）
3. `WiFi已连接` + `IP:192.168.x.x`（显示1秒）
4. `正在获取天气...`（首次拉取API，约1-2秒）
5. 进入正常显示界面（4行天气+时钟）

### 4. 日常运行

- 程序启动后无需干预，自动循环运行
- 天气数据每10分钟自动刷新
- 实时时钟每500ms刷新一次

## 显示效果

### 正常显示

```
上海实时天气
天气:晴 28C
22-30C 东南风
风力:3级 14:25:36
```

### 字段说明

| 字段 | 含义 | 数据来源 |
|------|------|---------|
| 天气:晴 | 当前天气状况 | now.json → now.text |
| 28C | 当前温度（摄氏度） | now.json → now.temperature |
| 22-30C | 今日最低温-最高温 | daily.json → daily[0].low/high |
| 东南风 | 今日主风向 | daily.json → daily[0].wind_direction |
| 风力:3级 | 今日风力等级 | daily.json → daily[0].wind_scale |
| 14:25:36 | 实时北京时间（HH:MM:SS） | NTP同步后的RTC |

## 常见问题

### Q1: 第4行时间显示 `00:01:41`，不是实时时间

**原因**：NTP未同步成功，掌控板开机后RTC从0开始计时。

**排查**：
1. 确认WiFi已连接（屏幕显示 `WiFi已连接`）
2. 检查串口输出是否打印 `NTP同步成功`
3. 若打印 `NTP同步失败`，检查网络是否能访问 `ntp3.aliyun.com`

**解决**：程序已包含主备两个NTP服务器自动切换，通常重启即可解决。若网络环境屏蔽NTP（UDP 123端口），需更换网络或使用HTTP时间API。

### Q2: 屏幕显示 `获取失败`

**原因**：心知天气API请求失败，可能原因：
- API密钥失效
- 网络无法访问 `api.seniverse.com`
- API免费额度用尽（每日有调用次数限制）

**排查**：查看串口打印的 `天气获取失败:` 后的具体异常信息。

### Q3: 天气数据不更新

**原因**：刷新间隔设置过长，或主循环异常。

**解决**：
- 检查 `shanghai_weather_oled.py` 中 `REFRESH_INTERVAL` 配置
- 查看串口是否打印 `主循环异常:`

### Q4: WiFi连接失败

**排查**：
1. 确认SSID和密码正确（区分大小写）
2. 掌控板仅支持2.4GHz WiFi，不支持5GHz
3. SSID不支持中文
4. 确认路由器DHCP功能开启

### Q5: 温度显示带小数点显示不全

OLED每行可显示约16个英文字符或8个中文字符。如温度数据过长可能截断显示，可调整 `shanghai_weather_oled.py` 中 `oled.DispChar` 的起始x坐标。
